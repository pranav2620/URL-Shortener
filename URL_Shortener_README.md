# URL Shortener

A web app that takes long URLs, generates short ones, tracks how many times each gets clicked, and lets you delete the ones you no longer need. Built with Node.js, Express, MongoDB, and EJS.

Nothing groundbreaking — but it does what it says, and it's a solid way to understand how the request-response cycle works end to end.

---

## What it does

- Paste a long URL, get a short one back
- Click the short URL — it redirects to the original and increments the click count
- Delete any URL you no longer want
- All URLs and click counts persist in MongoDB

---

## How it works

Three moving parts: the browser, the Express server, and MongoDB.

**Shortening a URL**
The user submits a form with the full URL. The server receives a POST request to `/shorturls`, generates a short ID using the `shortid` module, saves the document to MongoDB (with `full`, `short`, and `clicks` fields), and sends the updated list back to the page.

**Redirecting**
When someone visits a short URL, the server looks up the matching document, increments `clicks`, and issues an HTTP redirect to the original URL. If the key doesn't exist, it returns a 404.

**Deleting**
A DELETE request hits the server with the URL's document ID. The server finds it and removes it from the collection.

---

## Database schema

One MongoDB collection: `urls`

| Field | Type | Notes |
|---|---|---|
| `full` | String | The original URL. Required, unique. |
| `short` | String | Generated by `shortid`. Required, unique. |
| `clicks` | Number | Starts at 0, increments on each redirect. |

## Setup

**Prerequisites**
- Node.js — [download](https://nodejs.org/en/download/)
- MongoDB Compass — [download](https://www.mongodb.com/try/download/compass)
- VS Code or any editor

**Steps**

1. Clone the repo and open it in your editor
2. Open MongoDB Compass, connect to `mongodb://localhost:27017`
3. In the terminal, run:

```bash
npm install
npm start
```

4. Visit `http://localhost:5000`

## Known limitations

A few things worth knowing before using this in anything beyond local development:

- No input validation — the app doesn't check whether the submitted URL is actually a valid URL
- No authentication — anyone with access can delete any URL
- Single server setup — no load balancing, no redundancy
- Short codes generated by `shortid` are random but not cryptographically secure; they can theoretically be guessed

These aren't hard to fix and make for good next steps if you want to extend the project.

## What's next

A few directions worth exploring if you want to take this further:

- Add input validation and sanitization
- Rate limiting to prevent abuse
- User authentication so people can only manage their own URLs
- Custom aliases — let users pick their own short slug
- URL expiry with MongoDB TTL indexes
- A proper REST API with documented endpoints instead of form-based submission
